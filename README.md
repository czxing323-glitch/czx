引言与研究背景
	稀疏码分多址（SCMA）概述
稀疏码分多址（SCMA）作为一种多维非正交多址接入（NOMA）技术，通过将多维复星座图与稀疏稀疏图相结合，实现了在前向链路与反向链路上的高光谱效率和过载接入。在典型4*6SCMA系统中，4个正交的资源块（RE）可以同时承载6个不同的用户，系统的过载率达到了150%。接收端则利用消息传递算法（MPA）在因子图上进行多用户检测。然而，SCMA系统的整体误码率（BER）性能在很大程度上取决于多维码本的设计质量。
	现有拉丁生成矩阵设计的局限性
传统上，为了降低设计复杂度，常采用基于拉丁方（LatinSquare）的生成矩阵配合固定的等差离散角度相位差Δ。然而，人工指定的相位差难以保证所有叠加状态在复平面上的最优分布。具体而言，存在以下两大痛点：
（1）连续交织变量的非凸寻优难题：每一个非零资源块都需要一个独立的连续旋转角度θ。在4*6系统中，总共包含12个非零元素，对应12个相互交织的角度。这构成了一个典型的高维非凸优化空间，变量之间相互牵制，传统的集中式梯度下降法极易陷入局部最优。
（2）传统硬打分算子的“死锁”现象：传统设计往往以最大化最小欧氏距离（MED）为目标。但在初始阶段，由于等差角度的对称性，大量的复合星座点处于严重重合状态。此时，如果采用硬性的min算子，只要有一对点重合，总分就恒为0。这导致算法在微调某个单一角度时，由于无法在全局得分上体现出微弱的变好趋势，从而产生梯度阶跃，使整个寻优过程死锁。
	多智能体协同自助系统的引入意义
针对上述问题，本系统提出将12个交织的角度变量解耦为12个自主决策的角度优化智能体（AngleOptimizationAgents）。结合改进的负指数平滑距离求和打分法（Soft-SumMetric）作为协同演化的环境反馈，构建一个分布式协同自助优化系统。该系统不仅能自动探索最优的非对称酉变换旋转矩阵，还能有效打破初始重合带来的局部死点，兼顾了研一阶段的工程实现可行性与学术创新性。
系统总体架构设计
本自助系统采用“中心化协同评估，分布式自主微调”的二级多智能体架构。整个系统由物理仿真环境层、多智能体行为层和中央协同调度层三大核心部分组成。
 
	物理仿真环境层（SCMAEnvironmentLayer）
物理环境层是整个系统的基石，它负责模拟真实的通信物理链路。
（a）输入：由12个智能体输出的实时角度向量Θ=[θ_1,θ_2,…,θ_12 ]
（b）处理：环境层将这些连续角度分别代入二维酉旋转矩阵R(θ)=(■(cos⁡θ&-sin⁡θ@sin⁡θ&cos⁡θ ))中，替换掉基础拉丁生成矩阵中的非零元素。接着，将所有6个用户的多维基础星座进行相位旋转并执行空中叠加，最终在4个资源块上分别投影出包含64个复合主星座点的点集。
（c）输出：64个复合星座点的复平面坐标矩阵，供上层进行协同打分。
	多智能体行为层（Multi-AgentLayer）
多智能体层将复杂的12维联合搜索空间化整为零。
（a）智能体实例化：针对拉丁矩阵中的12个非零元素位置，分别实例化12个相互独立的AngleAgent。
（b）本地状态与局限性：每一个智能体只能感知和控制自己负责的单一角度θ_k。这种局部性极大地简化了单个智能体的决策复杂度。
动作行为：在被中央调度器激活时，智能体基于当前状态生成微调动作为探索策略。
	中央协同调度层（CentralCoordinationLayer）
由于12个角度在物理空间上是通过叠加相互干扰的，如果任由多智能体同时胡乱调整，会导致系统陷入混沌状态。因此，中央协同层起到了“指挥官”的作用。
（a）协同机制：采用轮询（Round-Robin）博弈机制。每次只允许一个智能体动作，其余11个智能体保持静止，以此解除变量之间的寄生交织耦合。
（b）奖惩决策：收集环境层叠加后的64个点集，利用核心创新打分算法评估当前微调的优劣，决定是否批准该智能体的本地状态更新。
多智能体协同机制与改进数学模型
	优化目标函数
系统的核心目标是通过优化旋转角度向量Θ，最大化接收端多用户叠加星座点之间的区分度，从而在低信噪比（SNR）下实现最低的误码率（BER）。在几何上，这等价于优化复合主星座点集X_"comp" 内任意两点之间的距离谱。
	传统硬打分算子分析（MED）
传统方案采用最小欧氏距离作为优化指标：
D_"min"  (Θ)=min┬(i≠j)⁡‖x_i-x_j ‖, x_i,x_j∈X_"comp" 
致命缺陷：当初始码本存在多点严重重合时，D_"min" =0。此时即使Agent1做出了一次非常优秀的微调，使得原本重合的10对点拉开了8对，但由于还剩2对重合，全局D_"min" 依然为0。协同层收到的反馈是“动作无效”，从而拒绝了该更新。这导致多智能体协作流产，系统死锁。
	核心创新：负指数平滑距离求和打分法（Soft-SumMetric）
为了给多智能体提供连续的、柔性的环境奖励反馈，本系统引入了负指数平滑距离求和函数作为协同层的核心评估指标：
S(Θ)=∑_(i=1)^(M-1)▒∑_(j=i+1)^M▒exp⁡(-‖x_i-x_j ‖^2/(2σ_0^2 )) 
其中M=64是复合主星座点的总数，$\sigma_0$是调节平滑敏感度的缩放因子。
数学物理机理：
（a）势能场模拟：该函数将每一个星座点视为一个“负电子”，点与点之间的靠近会产生指数级暴增的“斥力势能”。优化目标转换为最小化全局总势能S(Θ)。
（b）消除梯度阶跃：即使局部依然存在重合点（产生固定的高势能项），但只要某个智能体的微调拉开了其他未重合点的距离，总势能S(Θ)就会灵敏地下降。协同层就能捕捉到这一微弱的积极信号，从而批准该智能体的动作。这为多智能体演化开辟了一条连续的非线性滑行通道。

各文件功能详细生成说明
	config.py（参数配置文件）
生成功能：统一管理整个系统的硬件与算法超参数。
包含的关键参数项：
（a）SCMA资源配置：资源块数K=4，用户数J=6，每个资源块承载用户数V=3。
（b）智能体配置：微调搜索步长delta_theta=0.5（角度制），最大探索轮数MAX_EPOCHS=200。
（c）评估器配置：平滑因子sigma_0=0.1，打分器模式选择METRIC_MODE='SOFT_SUM'（可切换为'TRADITIONAL_MED'作对比实验）。
	agent/angle_agent.py（角度优化智能体类）
生成功能：生成具体智能体的行为模型。通过面向对象的方法，系统中将根据拉丁生成矩阵的结构，实例化12个完全独立的角度智能体实例。每个智能体维护自己负责的角度状态。
	agent/coordinator_engine.py（中央协同调度引擎）
生成功能：整个多智能体系统的“灵魂文件”。它生成两项核心功能：
（a）实现Round-Robin轮询调度器，控制12个智能体轮流“单挑”环境而不发生冲突。
（b）生成核心创新打分器（Soft-SumMetric），对环境反馈的叠加点集进行计算，为智能体提供奖惩决策。
	environment/constellation_env.py（物理映射环境）
生成功能：生成SCMA星座映射与叠加的物理空间。接收12维角度，生成替换酉矩阵，并将6个用户的QAM基础星座投射为4个资源块上的64个复合主星座点集。
核心方法：
（a）generate_latin_rot_matrix(angles)：将原拉丁生成矩阵中的1根据传入的12个角度，替换为(■(cos⁡θ&-sin⁡θ@sin⁡θ&cos⁡θ ))。
（b）get_combined_constellation(angles)：模拟多用户在资源块上的空中信号叠加，返回x_1 e^(jϕ_1 )+x_2 e^(jϕ_2 )+x_3 e^(jϕ_3 )后的64个主星座复平面坐标，作为打分器的直接输入。
	environment/link_evaluator.py（链路验证与译码器）
生成功能：生成下游学术价值验证与性能报告。当多智能体协同引擎输出最优的一组角度后，该文件负责将这一组角度注入真实的通信链路。通过模拟加性高斯白噪声（AWGN）信道，并在接收端运行消息传递算法（MPA译码器）。
核心方法：
（a）mpa_decoder()：在因子图上进行多用户概率解耦迭代计算。它会利用多智能体优化后的、毫无重叠的主星座点集计算指数似然：exp⁡(-‖y-∑▒h_m  x_m ‖^2/(2σ^2 ))。由于星座点被拉开了，条件概率的区分度暴增，译码置信度大幅度提升。
（b）plot_ber_curve()：自助生成并导出误码率（BER）对比曲线图（优化前固定等差角度的曲线vs多智能体协同优化后的连续非对称曲线）。
	main.py（全局自助运行总入口）
生成功能：系统的总控制台。用户在此文件执行一键启动。它依次加载config.py，实例化constellation_env.py的环境，创建12个智能体，将它们塞给中心协同引擎coordinator_engine.py开启寻优。寻优结束后，自动调用link_evaluator.py生成论文级别的BER性能图表。
各模块间的数据流与控制交互逻辑
为使系统正常运转，各文件之间的调用与数据传递必须遵循以下严谨的生命周期流程：
（1）配置加载阶段：main.py启动，首先从config.py导入全局参数。
（2）环境与智能体初始化：main.py调用constellation_env.py初始化物理层矩阵环境；随后根据环境矩阵维度，循环实例化12个angle_agent.py对象。
（3）协同演化大循环：main.py将环境和智能体列表作为参数传给coordinator_engine.py并启动协同器。协同器触发轮询机制，固定其他变量，单独激活某一个智能体进行±δ的微调。
（4）环境反馈与打分决策：智能体微调的角度传入constellation_env.py合成新的64个叠加点，协同器调用本地compute_soft_sum_score方法对点集进行负指数求和评估。若总势能下降，协同器向该智能体发送“批准”指令，智能体更新本地状态；否则发送“拒绝”指令，智能体回滚。
（5）链路最终验证：多智能体全面收敛后，main.py导出最优角度矩阵，并将其灌入link_evaluator.py，驱动接收端MPA译码器跑满10^5次蒙特卡洛仿真，最终打印并展示优化成果。
